# The `generate-identity` command

`zallet generate-identity` generates a new [age] identity for encrypting the wallet's key
material, and writes it to an identity file that [`zallet
init-wallet-encryption`](init-wallet-encryption.md) can consume. It uses the same `age`
library that Zallet links internally, so no external `rage` / `rage-keygen` binary is
required.

```
$ zallet generate-identity
Public key: age1...
```

## Output location

By default the identity is written to the configured `keystore.encryption_identity` path
(`encryption-identity.txt` in the data directory which is `~/.zallet` by default). The data directory can be overridden with `-d $DIRECTORY` and the output path and file can be specified with `-o/--output`.  Use `-o -` to write the identity to stdout, which may be useful for automated provisioning.

An existing identity file is never overwritten. To regenerate, delete the existing file
first — but beware that if a wallet has already been initialized with that identity,
deleting it makes the wallet's key material unrecoverable.

## Plain vs passphrase-encrypted identities

Without flags, a plain identity file is written, in `rage-keygen`'s format (a `# created:`
and `# public key:` comment header followed by the `AGE-SECRET-KEY-1...` line):

```
$ zallet -d /path/to/zallet/datadir generate-identity
Public key: age1...
```

With `-p/--passphrase`, the identity is passphrase-encrypted and ASCII-armored. In
interactive use you are prompted for the passphrase (with confirmation). In non-interactive
contexts (such as automated provisioning), the passphrase is read from the
`ZALLET_IDENTITY_PASSPHRASE` environment variable instead:

```
$ ZALLET_IDENTITY_PASSPHRASE=... zallet -d /path/to/zallet/datadir generate-identity -p
Public key: age1...
```

The environment variable, when set, is read once and is not persisted by Zallet.

## Plugins

age plugin identities (e.g. YubiKey, Apple Secure Enclave, OpenPGP card) require the
corresponding age plugin binaries and are not generated by this command. See
[`init-wallet-encryption`](init-wallet-encryption.md) for using plugin identities.

[age]: https://age-encryption.org/
